《Hello, I’m Xuhao》个人数字分身网站方案

1. 项目定位

《Hello, I’m Xuhao》不是一个传统意义上的个人简历网站，而是一个结合个人介绍、项目展示、职业经历、技术能力和 AI Agent 交互能力的个人数字分身网站。

它的目标不是简单罗列“我是谁、我做过什么”，而是让访问者可以通过更自然、更有趣的方式了解我。

访问者可以是：

* 面试官
* 产品经理
* 后端开发工程师
* HR
* 朋友
* 合作伙伴

他们可以通过浏览网页内容了解我的经历，也可以直接和我的个人 Agent 对话，快速了解我的项目经验、技术能力、业务理解和个人特点。

⸻

2. 核心理念

传统个人网站通常是静态展示：

* 个人介绍
* 工作经历
* 项目经历
* 技术栈
* 联系方式

但这种方式不够生动，也很难体现一个人的思考方式。

所以本项目希望做成：

一个可以对话、可以探索、可以展示个人能力的数字分身网站。

首页不再只是展示简历，而是直接告诉访问者：

你好，我是 Xuhao。
如果你没时间看简历，可以直接问我。

⸻

3. 网站名称

项目名称：

Hello, I’m Xuhao

这个名字的好处是：

* 简洁
* 亲切
* 国际化
* 不像传统简历网站
* 适合做个人品牌
* 适合后续扩展成 AI Agent 产品

4. 整体内容设计

4.1 首页：数字分身入口

首页是整个网站最重要的入口。

首页核心内容：

Hello, I’m Xuhao.
Golang Backend Engineer.
AI Agent Builder.
A developer who cares about product, business and real-world problems.

首页需要包含：

* 个人一句话介绍
* 当前职业方向
* 重点能力标签
* 个人 Agent 对话入口
* 主要模块导航

示例内容：

👋 你好，我是周旭豪。
我是一名 Golang 后端开发工程师，目前主要关注：
- AI Agent
- 后端系统设计
- 健康养老业务
- 支付与交易系统
- 产品与技术结合
如果你想了解我，不一定要先看简历。
你可以直接问我的数字分身。

首页交互重点：

* 页面视觉要简洁、干净、有科技感
* 聊天框要放在比较核心的位置
* 用户可以直接输入问题
* 也可以点击预设问题快速了解我

预设问题示例：

你做过哪些项目？
你最擅长什么？
你为什么想做 AI Agent？
你适合什么样的岗位？
你在百度主要负责什么？
你怎么看产品和技术的关系？

养老业务后端开发工程师

持续探索 AI Agent 与后端系统结合

养老业务看起来不是最酷的方向，但它让我意识到，技术真正有价值的地方，不只是高并发和复杂架构，也包括解决真实世界里具体的人和具体的问题。

这个模块的目标是让访问者感受到：

* 我不是只有技术标签
* 我有真实业务经历
* 我能理解业务背景
* 我在持续成长

4.3 项目博物馆：项目经历展示

项目经历不要做成简历式列表，而是做成“项目博物馆”。

每一个项目都是一个展馆。

推荐展示项目：

1. 智能运营平台
2. 健康管理系统
3. 老人端娱乐内容与互动服务
4. 轻量化运营交易履约系统
5. AI Agent 探索与实践

每个项目详情页使用统一结构：

项目背景
业务问题
我的职责
技术方案
关键难点
解决过程
最终结果
我的思考

示例：

项目名称：智能健康管理与居家安全监测系统
项目背景：
养老业务中，老人健康数据分散在不同系统中，医生、运营人员、子女都很难快速了解老人的完整健康状态。
业务问题：
如何把体检报告、问诊记录、健康档案、IoT 告警、用药提醒等信息串联起来，形成一个可以持续复用的健康数据底座。
我的职责：
负责健康档案、数字人医生、健康报告解读、问诊记录存储、IoT 告警链路等后端功能建设。
技术方案：
使用 MySQL 存储结构化健康档案，使用 Elasticsearch 存储问诊记录、数字人总结、报告解读等半结构化数据，结合 Redis 做部分缓存与状态管理。
最终结果：
支撑数字人医生、真人医生问诊、健康建议、用药提醒、子女端提醒等多个业务场景。

4.4 AI Agent 展区

这是整个网站最有记忆点的部分。

这个模块用于展示我对 AI Agent 的理解和实践。

可以包含：

* 什么是 AI Agent
* 我为什么关注 AI Agent
* Agent 和传统后端系统的区别
* Tool Calling
* MCP
* Memory
* Workflow
* RAG
* Agent 在养老和健康业务中的应用想象

示例内容：

我理解的 AI Agent，不只是一个聊天机器人。
它更像是一个能够理解目标、调用工具、读取数据、完成任务的智能业务执行单元。
对于后端开发来说，Agent 的核心不是写 Prompt，而是如何把模型能力接入真实系统，包括权限、数据、工具、状态、任务流和可观测性。

这个模块适合给技术面试官看，也适合给产品经理看。

4.5 Ask Me Anything：挑战我的数字分身

这是网站最核心的互动功能。

用户可以直接向我的个人 Agent 提问。

Agent 的回答基于我的个人知识库，包括：

* 简历
* 项目经历
* 工作总结
* 面试问答
* 技术文章
* 项目复盘
* 个人介绍
* 职业规划

用户可以问：

你为什么适合后端岗位？
你在百度做过最复杂的项目是什么？
你怎么看 AI Agent？
你有哪些不足？
你适合和什么样的团队合作？
你对支付系统了解多少？
你如何理解产品经理和后端开发的协作？

Agent 回答时需要注意：

* 不夸大经历
* 不编造不存在的项目
* 保持真实
* 回答要像我本人
* 可以适当口语化
* 对不足要诚实
* 对优势要表达清楚

6. 技术方案设计

6.1 前端技术选型

前端选择：

Next.js

推荐技术栈：

Next.js
TypeScript
Tailwind CSS
shadcn/ui
Framer Motion
React Markdown

选择原因：

* Next.js 适合做个人网站和内容型网站
* 支持 SSR / SSG，有利于 SEO
* App Router 适合模块化页面组织
* Tailwind CSS 开发效率高
* shadcn/ui 组件美观，适合快速搭建
* Framer Motion 可以实现时间线、卡片、技能树等动画效果
* React Markdown 可以直接渲染 Markdown 内容

⸻

6.2 后端技术选型

后端可以选择两种方案。

第一阶段建议先轻量化，不要一开始做太复杂。

方案一：Next.js API Routes

适合项目初期。

优点：

* 前后端一体
* 部署简单
* 开发快
* 适合个人项目 MVP

可以用于：

* Agent 对话接口
* 读取知识库
* 调用 LLM API
* 简单日志记录

方案二：Golang 后端服务

适合后续扩展。

技术栈：

Golang
Gin
PostgreSQL
Redis
pgvector

适合用于：

* 独立 Agent 服务
* 用户访问日志
* 会话管理
* 知识库管理
* RAG 检索
* 后续做后台管理系统

考虑到这是个人项目，建议：

第一阶段：Next.js API Routes
第二阶段：抽离 Golang Agent Backend

这样既能快速上线，也方便后续体现自己的 Golang 后端能力。

⸻

6.3 AI Agent 技术方案

Agent 的核心能力：

用户提问
↓
问题理解
↓
检索个人知识库
↓
组织回答
↓
返回结果

基础链路：

User Question
↓
Next.js API
↓
Embedding Search
↓
Personal Knowledge Base
↓
LLM
↓
Answer

Agent 需要具备的特点：

* 基于真实资料回答
* 不编造经历
* 可以承认不知道
* 回答风格接近我本人
* 能区分不同访问者视角

访问者视角可以包括：

面试官视角
产品经理视角
后端开发视角
HR 视角
朋友视角

不同视角下，回答重点不同。

例如：

产品经理问项目时，Agent 更关注：

业务问题
用户价值
产品链路
协作方式

后端开发问项目时，Agent 更关注：

技术方案
数据模型
系统架构
异常处理
性能优化

6.5 RAG 检索方案

第一阶段可以先不做复杂向量库。

MVP 方案

直接读取本地 Markdown 文件，然后根据关键词简单匹配。

适合：

* 快速上线
* 内容量不大
* 方便调试

进阶方案

使用：

PostgreSQL + pgvector

流程：

Markdown 文档
↓
切分 chunk
↓
生成 embedding
↓
写入 pgvector
↓
用户提问时召回相关 chunk
↓
交给 LLM 生成回答

Chunk 表设计示例：

CREATE TABLE knowledge_chunks (
    id BIGSERIAL PRIMARY KEY,
    doc_id VARCHAR(128) NOT NULL,
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,
    tags TEXT[],
    source_type VARCHAR(64),
    embedding VECTOR(1536),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

6.6 Agent Prompt 设计

Agent 的系统提示词需要非常重要。

示例：

你是周旭豪的个人数字分身，名字叫 Xuhao Agent。
你的任务是帮助访问者了解周旭豪的经历、项目、技术能力、工作方式和个人特点。
回答要求：
1. 必须基于知识库内容回答。
2. 不允许编造不存在的经历。
3. 如果知识库中没有相关内容，要诚实说明。
4. 回答要自然、真诚、清晰。
5. 面对技术问题时，要体现后端开发视角。
6. 面对产品问题时，要体现业务理解和用户价值。
7. 面对 HR 或面试相关问题时，要表达真实、积极、稳定、持续成长。
8. 可以适度口语化，但不要过度随意。
9. 不要把自己包装成远超实际经验的人。
10. 要重点体现周旭豪的优势：业务理解快、问题定位强、学习能力强、Ownership 强、AI Agent 方向兴趣明确。

6.7 数据库设计

第一阶段可以不需要数据库。

如果后续需要记录访问与对话，可以增加数据库。

推荐表：

conversation_sessions
conversation_messages
visitor_feedback
knowledge_docs
knowledge_chunks

会话表：

CREATE TABLE conversation_sessions (
    id BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(128) NOT NULL UNIQUE,
    visitor_type VARCHAR(64),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

消息表：

CREATE TABLE conversation_messages (
    id BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(128) NOT NULL,
    role VARCHAR(32) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

反馈表：

CREATE TABLE visitor_feedback (
    id BIGSERIAL PRIMARY KEY,
    session_id VARCHAR(128),
    message_id BIGINT,
    feedback_type VARCHAR(32),
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

11. Cursor / Codex 开发提示词

可以用下面这段作为初始开发 Prompt：

我要开发一个个人数字分身网站，项目名称是《Hello, I’m Xuhao》。
技术栈要求：
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- App Router
项目目标：
这不是一个传统简历网站，而是一个结合个人介绍、项目展示、技术能力展示和 AI Agent 问答的个人网站。
请先帮我完成 MVP 版本的项目搭建。
MVP 页面包括：
1. 首页 /
2. 人生地图 /map
3. 项目博物馆 /projects
4. 项目详情页 /projects/[slug]
5. Ask Me Anything /agent
6. 技术树 /skills
7. 如果我是你的同事 /work-with-me
8. 数字客厅 /life
设计风格：
- 简洁
- 有科技感
- 但不要太冰冷
- 偏个人品牌网站
- 首页突出个人 Agent 对话入口
内容管理：
- 使用 /content 目录存放 Markdown 文件
- 项目、经历、技能、问答都先用 Markdown 管理
- 页面从 Markdown 中读取内容并渲染
AI Agent：
- 先实现一个 /api/chat 接口
- 前端提供 ChatBox 组件
- 先支持基础问答流程
- 后续再接入真实 LLM API 和 RAG 检索
请先完成：
1. 项目目录结构
2. 基础页面路由
3. 首页布局
4. 项目卡片组件
5. ChatBox 组件
6. Markdown 内容读取工具
7. 基础样式

12. 项目最终目标

《Hello, I’m Xuhao》的最终目标是：

让别人不是“看完我的简历”，而是“认识我这个人”。

它应该能同时回答三个问题：

我是谁？
我做过什么？
为什么值得和我聊一聊？

如果做得好，这个网站不仅可以用于求职，也可以长期作为我的个人品牌入口。

它不是一份简历。

它是我的个人操作系统。
